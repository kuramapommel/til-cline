# UI 情報の抽出・記録ガイド（Step 2 / Step 8）

画面イメージからは、機能だけでなく **画面レイアウト・カラーパレット・コンポーネント構成** といった UI 情報が読み取れる。従来これらはストーリーに残らず捨てられていたが、これらは後続の実装・リファインメントで価値を持つため **破棄せず記録する**。

## 基本方針

1. **読み取れる UI 情報は文章で記載する**（`UI / 画面情報 (UI Notes)` セクション）。
2. **文章で表現しきれない情報**（微妙な色味・余白の取り方・図版でしか伝わらないレイアウト等）は、**参照した画像データそのものをストーリーのそばに置いて参照**する（後述の画像フォールバック）。
3. **創作しない**。画像から確実に読み取れる範囲だけを書く。曖昧なものは「〜と思われる」と明示するか、Step 6 の質問シートに回す。

## 抽出する UI 情報の観点

各画面イメージから、少なくとも次を読み取れる範囲で抽出する。

### レイアウト / 画面構成
- 全体構造（ヘッダ / サイドバー / メインエリア / フッタ 等の領域分割）
- 主要領域の配置関係（左右 2 カラム、上下スタック、中央寄せ 等）
- グリッド・カラム数、コンテンツ幅の印象（全幅 / 中央固定幅 等）
- レスポンシブの示唆があれば（PC/SP 両方のカンプがある等）

### カラーパレット
- 背景色 / サーフェス色 / 文字色 / アクセント色 / 罫線色
- 状態色（成功・警告・エラー・無効）が読み取れれば併記
- 可能なら **HEX 値または近似色名**まで踏み込む（スポイト的に読み取れる範囲で）。断定できなければ「濃紺系」「淡いグレー」等の色相・トーン表現に留める

### タイポグラフィ（読み取れれば）
- 見出し / 本文の大小関係、フォントの印象（ゴシック / 明朝、太さ）

### 主要コンポーネントと状態
- ボタン（プライマリ / セカンダリ、活性 / 非活性）
- 入力欄（テキスト / スライダー / セレクト / チェック / ラジオ）とプレースホルダ・初期値
- リスト / テーブル / カード / モーダル / トースト等
- 各コンポーネントの状態（選択中・エラー表示・ホバー等）が読み取れれば添える

## 記録先（ストーリーへの反映）

- 抽出した UI 情報は、対象ストーリーの `## UI / 画面情報 (UI Notes)` セクションに箇条書きで書く。
- そのストーリーに直接関係する範囲に絞る（1 画面から複数ストーリーが起きた場合、各ストーリーには**そのストーリーが対象とする UI 要素**を中心に書く。画面全体に共通するシェル情報は関連する全ストーリーに簡潔に添えてよい）。

## 画像フォールバック（文章化できないとき）

文章化しきれない UI 情報がある場合は、**参照した画像データをストーリーのそばに配置**して Markdown から参照する。

### 配置場所

```
docs/userstory/<story-file-basename>/<image-file-name>
```

- `<story-file-basename>` は、そのストーリーの Markdown ファイル名から拡張子 `.md` を除いたもの。
  - 例: ストーリー `V1StGXR8a9_training-list_view.md` に対し `docs/userstory/V1StGXR8a9_training-list_view/` を作る。
- `<image-file-name>` は元画像のファイル名（安全化: 空白や禁止文字 `/ \ : * ? " < > |` は `-` に置換）。

### 参照の書き方（相対パス）

ストーリー Markdown（`docs/userstory/` 直下）からの相対パスで参照する。

```markdown
- 参照画像: ![研修一覧画面のカラーとレイアウト](./V1StGXR8a9_training-list_view/screen.png)
```

### 画像の配置手段

- **ローカルパス / コピー可能なファイル**として入力が渡された場合は、`cp` 等で上記ディレクトリへコピーする。
- 元画像が **チャットに貼り付けられただけでファイル実体を取得できない**、URL からダウンロードする手段が無い等でコピーできない場合は、無理に配置しない。
  - その旨を明示し、ユーザーに「画像ファイルを `docs/userstory/<story-file-basename>/` に置いてください」と依頼するか、読み取れた範囲を最大限文章で `UI Notes` に書き残す。
- 同一画像が複数ストーリーで必要な場合は、各ストーリーのディレクトリにコピーしてよい（フォールバックとして発生する頻度は低い想定）。重複を避けたい場合は代表 1 ストーリーのディレクトリに置き、他ストーリーから相対パスで参照してもよい。

## Step 6（質問シート）との接続

読み取れなかった / 自信の持てない UI 情報（例: 「アクセント色は青系だが正確な値は不明」「この余白が意図的か不明」）は、Step 6 の質問シートに「UI 確認」項目として回してよい。ただし UI の細部は本スキル（DRAFT 抽出）の主眼ではないため、**ストーリー確定に必要な範囲**に絞ること。
