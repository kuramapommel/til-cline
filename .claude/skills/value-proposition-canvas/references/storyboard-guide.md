# UX ストーリーボードの考え方と表現指針

成果物 HTML の (B) UX ストーリーボード部の作り方。

## UX ストーリーボードとは

製品・サービスを利用したユーザーの**理想体験**を、4 コマ漫画や絵コンテのような連続するシーンで表現する手法。文字だけのフロー説明より、**感情の流れと文脈**が伝わりやすい。

## 表現の自由度

**テンプレ化しない**。導出した Products and Services のイメージに合わせて、毎回自由に表現する。

- 4 コマ（起承転結 4 シーン）
- 絵コンテ（5 〜 8 コマで体験のヤマを描く）
- ビフォー / アフター 2 コマ（変化の対比を強調する場合）
- タイムライン横長型（時間軸に沿って体験の流れを描く）
- 状態遷移型（感情グラフ＋シーン）

「どれが正解」はない。**そのペルソナと製品・サービスにとって最も体験が伝わる形**を選ぶ。

## SVG での描き方

`persona` のアバターと同様、**丸・四角・線・色で表現できるかなりデフォルメされたもの**でよい。写実性は不要。

各シーンの構成要素:

- **人物**: 顔は単純化（円＋目＋口の線）。表情でストーリーを語る（困り顔・驚き顔・笑顔）。
- **環境**: 1 〜 3 個のオブジェクトで場所を示唆（PC、机、椅子、窓、スマホ、ベッド）。
- **吹き出し / 内心**: 感情やセリフを端的に表現。長文にしない。
- **キャプション**: コマ下にシーンを 1 行で説明（〜10 文字目安）。

## 構成のコツ

- **起点となる Pain を最初のコマに置く**: 現状の困りごと・嫌な瞬間。
- **製品・サービスとの出会い**を 2 コマ目に。
- **使ってみたときの体験**を中盤に。
- **得られた Gain（情緒的な変化を含む）**を最後のコマに。

## 例: 4 コマ構成

```
[コマ1] PC前で集中したいのに通知が次々鳴る、困り顔
        キャプション「集中したい朝。なのに通知の波。」

[コマ2] アプリの "集中モード" を起動、安堵の表情
        キャプション「集中モードでバンドル開始。」

[コマ3] 静かな環境で実装に没頭、笑顔
        キャプション「90分、誰にも邪魔されず実装完了。」

[コマ4] 夕方、PC を閉じて窓の外を見ている、満足顔
        キャプション「定時で終わって、夜は自分の時間。」
```

## SVG の技術的なヒント

- 1 コマ ≒ 240 × 180 px 程度がバランスが良い。横並びで 4 コマ ≒ 1000 × 200 px。
- 顔: `<circle>` で輪郭、`<circle r=2>` × 2 で目、`<path>` で口（笑顔は M-A、困り顔は M-V）。
- 表情の差分だけで感情の流れが描ける。
- 配色は 2 〜 3 色に絞る（背景色 + アクセント 1 色 + テキスト色）。
- フォントサイズはキャプション 12px、吹き出し 11px 程度。

## ストーリーボードを描くタイミング

ステップ 5 の HTML 生成時。Products and Services が確定し、Fit 点検が済んだ後に描く。

ストーリーボードを描いたとき、エージェントが「この体験は本当にこのペルソナに刺さっているか?」を最終チェックすることになるので、**Fit 点検の自然な確認役にもなる**。違和感があれば、HTML 出力前にユーザーに気づきとして返してよい。
