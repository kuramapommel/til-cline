# バックログデータベース作成プロンプト

「テストプロダクト」ページの直下に、開発タスクを管理するためのバックログ用データベースをインラインデータベースとして新規作成してください。

## データベース名

バックログ

## 作成するプロパティ（列）

以下のプロパティを持たせてください。

| プロパティ名 | 種類       | 設定                                                                                                                                                       |
| ------------ | ---------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ID           | ID         | バックログ項目に一意のバックログ ID を自動採番する。GitHub PR との紐付けに使用する。プレフィックスは GitHub Pull Requests プロパティの手動設定時に設定する |
| 名前         | タイトル   | バックログ項目のタイトル                                                                                                                                   |
| ステータス   | ステータス | 選択肢を「未実装」「実装中」「実装済み」の3つにする。デフォルトは「未実装」                                                                                |
| 関連 Issue   | URL        | GitHub Issue へのリンク。初期は空                                                                                                                          |
| 作成日       | 作成日時   | 自動記録                                                                                                                                                   |

## 各項目（行）の本文に持たせたいテンプレート

各バックログ項目を開いたときに、本文に以下の見出し構成がテンプレートとして入るようにしてください。

- 背景 / Context
- ゴール / Goal
- 受け入れ基準 / Acceptance Criteria
- スコープ外 / Out of Scope
- 関連仕様書（同じ「テストプロダクト」配下の既存ページ（01 PRD、03 機能仕様 など）へのメンションリンクをここに記載する）

## ビュー

以下の2つのビューを作成してください。

1. テーブルビュー: 「未実装」のみを表示するフィルタ付きのテーブル（デフォルトビューとして最初に作成すること）
2. ボードビュー: ステータス（未実装 / 実装中 / 実装済み）ごとに列を分けたカンバン表示

## 補足

- 既存の 01〜07 のページ群やその内容は一切変更しないでください。あくまでバックログ用データベースを新規追加するだけです。
- 関連仕様書は既存ページへのメンション（本文リンク）で緩く紐付ける運用とします。新しくリレーション用のデータベースを作る必要はありません。
- バックログ ID は GitHub PR との紐付けに使用します。PR タイトルにバックログ ID を含めることで、Notion が対象のバックログ項目を自動で特定します。

## 完了後の案内

データベースの作成が完了したら、以下の手動設定が必要である旨をユーザーに伝えてください。手順の詳細は、あなた（Notion AI）が現在の Notion の UI に合わせて生成してください。

**手動設定1: GitHub Pull Requests プロパティの追加**

作成したバックログデータベースに「GitHub Pull Requests」プロパティを追加する。これは Notion の GitHub 連携で提供される専用のプロパティタイプであり、通常のテキストや URL プロパティではない。GitHub 連携が未設定の場合は、先に Notion ワークスペースと GitHub の連携を済ませる必要がある。追加時にバックログ ID のプレフィックスを「PBI」に設定し、PBI-1, PBI-2 形式で採番されるようにする。

**手動設定2: Auto-update の設定**

追加した GitHub Pull Requests プロパティに対して、PR がマージされたときにバックログのステータスを「実装済み」に自動変更する設定を行う。他の PR 状態（Opened / Review requested / Approved）は設定不要。
