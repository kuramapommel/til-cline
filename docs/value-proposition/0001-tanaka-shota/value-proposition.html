Value Proposition Canvas — Problem-Solution Fit までの仮説
田中 翔太（仮）— 28歳・FE エンジニア HYPOTHESIS

「集中している側を弱者扱いしない場」を提供することで、自律的なタイミングで繋がれる日常を取り戻す価値提案仮説。

提供者側 Value Map

どう価値を作るか

うれしさを生み出すものGain Creators

  • G2·G7GC-1. 「自律的なタイミングで繋がる」文化の浸透 罪悪感なく "今は集中" を宣言でき、相手側もそれを尊重する規範。M1 直交マッチからの導出。
  • G2·G3GC-2. 言葉にしない意図が伝わる安心感 「あとで返したい」「緊急じゃない」「気配だけ伝えたい」を硬いラベル無しで相互伝達。M2 転用マッチ。
  • G8GC-3. 集中文化を育てるコミュニティ／プラットフォーム プロダクト利用そのものが文化への参加となる。組織のコミュニティ運営力を活用。M3 転用マッチ（V2 以降の長期構想）。
  • G1·G4·G5GC-4. 終了時ダイジェストによる完遂感 集中時間の終わりに、本当に必要な情報・通知だけが整理されて届く。"取りに行かない" 状態が正常。M5 直交マッチ。

悩みを和らげるものPain Relievers

  • P1PR-1. 送信側に「集中している相手を弱者扱いしない」規範を伝える UI P1 構造的原因「即時応答強要×送受信非対称」に直接作用。送信時に「相手は集中中、応答がないことに罪悪感は要らない」と気づかせる介入。組織のビジョン「コミュニケーションが苦手な人を弱者扱いしない場づくり」の直交転用。最も深いレイヤー。
  • P1·P3PR-2. 気配を流す UX（言葉にしない意図の相互伝達） 「いま集中」「あとで返す」「緊急じゃない」を硬いラベルではなく気配として送受信。組織の暗黙知（言葉にしない意図を UX に翻訳する力）の直接転用。
  • P2·P4PR-3. 集中時間中は情報を "なくす" 設計＋終了時ダイジェスト P2 の根「通知の不確実性が認知資源を消費」を、不確実性そのものを取り除いて解消。情報統合の常識を逆転し「取りに行かないことが正しい」を実装。M5 直交。
  • P5PR-4. テキスト主体・非同期主体の MTG モード（V2） 既存プロト「声を出さない MTG」の再起動。書く方が深く考える内向型ユーザーが本当の課題を surface できる場。M4 直接マッチ（V2 統合）。

製品・サービスProducts and Services

  • Quietude（仮称）: SaaS + ブラウザ拡張 + Slack アプリ のハイブリッド 受信側体験（情報を "なくす"）／送信側規範形成（M1 の核）／チームへの自然な広がり、を 1 プロダクトで担う構成。
  • SaaS — 受信側のセッション管理・ダイジェスト生成・気配の蓄積
  • ブラウザ拡張 — Slack Web 上で送信側 UI に介入し「弱者扱いしない規範」を visualize
  • Slack アプリ — ステータス自動連動・気配の流通
  • 料金: フリーミアム（個人無料、チーム機能のみ有料） 「個人の集中体験を起点にチームに広がる」M3 の長期視点と整合する設計
  • 長期構想: 集中文化コミュニティ（V2 以降）／非同期 MTG モード（V2）

顧客側 Customer Profile

何が起きているか

顧客の用事Customer Jobs

  • J1. 細切れの時間でも集中して実装を進める 機能的
  • J2. 他チームと毎日連携しつつ、自分の作業計画を守る 機能的
  • J3. 自分のペースを保って働く 情緒的
  • J4. 同僚から「助かった」と感謝される関係性を作る 社会的
  • J5. 仕事を定時で終わらせて、夜は趣味に集中する 情緒的

うれしさGains

  • G1. 完遂感（PR がスムーズにマージ、難しい実装も気持ちよく）
  • G2. 自律性（自分のペースで決められる感覚）
  • G3. 同僚からの直接的な感謝
  • G4. 集中時間の確保による「挑戦できる」感覚
  • G5. 待ちからの解放（自分のスケジュールで完結）
  • G6. キャリアの充実した積み重ね 要検証
  • G7. オフの完全な回復
  • G8. チームへの広がり（想定外利得） 外部条件

悩みPains

  • P1. Slack の急な割り込みで集中が断ち切られる
  • P2. 半集中状態で疲弊（「次が来るかも」が気になる）
  • P3. 他チームのペースに引きずられて計画が侵食される
  • P4. 情報が Slack/Notion/GitHub/Jira に散在し探す時間が発生
  • P5. 表面的な MTG ばかりで本当の課題が見えない 部分カバー
  • P6. 組織の将来不安・「ここに居続けるべきか」 スコープ外
  • P7. 個人では改善できないという無力感 スコープ外

確信度

★★★☆☆

顧客側 3 ブロックは入力済みのペルソナ仮説に基づいており確度は中〜高。提供者側はペルソナ非依存の棚卸しから surface した内容で、Synthesis のメカニズムは論理的に通っている。ただし市場検証は未実施。とくに直交マッチ M1（送信側規範形成）は思想として強いが、実証は要 PoC。

検証ポイント

  • 送信側規範の浸透が、実際に発信抑制に作用するか（M1 の核） 差別化の核。「集中している人を弱者扱いしない」という規範が、一度の表示で送信側に内面化されるか、繰り返しの介入が必要かを PoC で確かめる。最優先。
  • 「気配を流す UX（PR-2）」の表現粒度 ラベル化と気配の境界。「いま集中」を硬く表すと FOCUS の二の舞、柔らかすぎると伝わらない。表現の言語化は要ユーザーテスト。
  • 情報を "なくす"（PR-3）の心理的受容 「届いていないかも」という不確実性が、別種の認知負荷を生まないか。終了時ダイジェストへの信頼が前提条件。
  • P5・P6・P7（組織課題）はカバー外。組織アプローチとの併用が必要かどうか PR-4（V2 非同期 MTG）が P5 に部分的に作用するが、P6・P7 は構造的に届かない領域。
  • G6（キャリア充実の積み重ね）に直接届く Gain Creator が無い 間接効果（GC-4 の完遂感が積み重なる）に留まる可能性。直接の振り返り機能が必要かは要検証。
  • G8（チーム広がり）が必要条件か単なる嬉しさか 必要条件なら導入戦略を「個人→チーム」から「チーム一括」に変える必要がある。

収益化の仮説（軽い案）

  • フリーミアム ＋ チームプラン 個人無料、チーム機能（規範形成の組織展開・集中文化スコア・ダイジェスト共有）はシート単価 800〜1,500 円／月。M3 の思想と整合。
  • 法人プラン（年契約） 中規模以上のチーム向け。組織レベルの規範形成支援・SSO・管理機能込み。
  • コミュニティ会費 ＋ プロダクト無料（V2 以降） 集中文化コミュニティの参加費が主収益、プロダクトは入口として無料。組織の長期ビジョンと整合。
  • 思想の出版・講座化 「集中している側を弱者扱いしない場」という思想を本・講座にして横展開。プロダクトのブランディングに寄与。
